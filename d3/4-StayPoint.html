II. Detection des Stay Points

Pour pouvoir dtecter les modes de transports sur les trajectoires, il faut dans un premier temps dtecter les phases "En mouvement" et les phases "Immobile".

Nous nous sommes donns pour objectif de dtecter les stay-point, c'est dire les points pour lesquels l'utilisateur est immobile.

A. Dtection de points d'intrt

Objectif : on veut partitionner une journe donne avec deux types de segments :

  • segment "en mouvement"
  • segment "immobile"

Algorithme #1Â¶

Notre premire intuition tait d'utiliser la vitesse pour cette segmentation, avec un rgle du type

En mouvement : vitesse > threshold
Immobile : vitesse < threshhold

Cependant, cet algorithme donne de trs mauvais rsultats, car le bruit, mme pour une trajectoitre filtre est trop important. (Cf Notebook prcdent avec le profil des vitesses)

Cette fois, on s'appuie sur la dentit des positions pour dtecter le mouvement ou non.

L'ide est que si des points conscutifs sont suffisamment proche les uns des autres, l'utilisateur est l'arret. La figure suivante illustre l'algorithme qu'on a imagin :

# Distance between two filtered points
def fdistance(df, i, j) :
    return distance.haversineDistance(
        df["lng_mean_filt"][i],
        df["lat_mean_filt"][i],
        df["lng_mean_filt"][j],
        df["lat_mean_filt"][j])

lower_limit = 3
radius = 50
i = 1790 # This is during a mouvement
start_stay_point = -1

while i < (day_df["timestampMs"].size - lower_limit - 1):
    
    # Vrifier si i est un dbut de stay point
    mouvement = False
    for k in range(lower_limit) :
        if (fdistance(day_df, i, i + k + 1) > radius) :
            mouvement = True

    if mouvement :
        # Si on est en mouvement, regarder le point suivant
        i += 1
    else :
        # Premier point immobile
        start_stay_point = i
        print("First Stay point : " + str(i));
        break;

La 3eme tape est de dtecter la fin du Stay Point.

On ajoute tous les points tant qu'ils sont  une distance infrieure  radius du point i.

Si k + 1 outliers sont dtect de suite, c'est la fin du Stay Point.

max_outliers = 5

        outliers -= 1

    else :
        outliers = max_outliers
    j += 1

# Fin du stay_point

IV. Filtrage par segment temporel

Les filtres mean filter et median filter utiliss jusqu' prsent s'appliquaient l'ensemble des points. Or le relev des points par le GPS peut s'arrter pendant un temps suffisamment long pour que des points successifs n'aient pas de rapport direct entre eux, et donc ne devraient pas s'influencer lors du filtrage. Par exemple si l'utilisateur teind son GPS pendant une heure, il n'est pas raisonable d'utiliser les points juste avant l'arrt et juste aprs la reprise comme s'il s'agissait d'un mme trajet continu.

Pour pallier ce problme, on va utiliser l'algorithme de segmentation par seuil de dlai entre deux points successifs prsent dans le notebook 2-segmentation. Cette fois on utilise un seuill plus grand de 400 secondes. Ce choix est motiv par l'observation des dlais sur plusieurs chantillons o il semble que lorsque les cas dpassant les 400 secondes sont plus isols et donc plus discriminant de diffrentes sessions d'activit.

Les filtres seront alors appliqus indpendamment chaque segment, cela nous permet d'utiliser les donnes du segment dans son ensemble pour choisir le filtre le plus appropri. Par exemple on peut se baser sur la densit spatiale des points ( savoir la distance moyenne entre deux points conscutifs) pour choisir le filtre. En effet les filtres mean filter et median filter ne sont efficaces que pour des trajectoires avec une bonne densit de points. On peut galement utiliser cette information pour rgler les paramtres du filtre : ici la taille de la fentre.

On teste donc ce filtrage par segment temporel avec les paramtres suivants :

- seuil de 400 secondes entre deux segments \n
- si la distance moyenne entre deux points sur un segment est infrieure  20 mtres, on applique un median filter avec une fentre de taille 7
- si la distance moyenne entre deux points sur un segment est entre 20 et 50 mtres, on applique un median filter avec une fentre de taille 5
- si la distance moyenne entre deux points sur un segment est entre 50 et 150 mtres, on applique un mean filter avec une fentre de taille 5
