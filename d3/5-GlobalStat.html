III. DBSCAN clustering sur les segments

Nous nous proposons ici d'appliquer l'algorithme DBSCAN pour agglomrer les diffrents trajets au cours d'une journe.

La segmentation de la journe en trajets sera ralis en utilisant l'approche prsente dans le notebook 3.

A. Principes de DBSCAN et prparation des donnes

DBSCAN (density-based spatial clustering of applications with noise) est un algorithme de partitionnement de donnes bas sur la densit des clusters pour effectuer le partitionnement.

Deux paramtres principaux prendre en compte par DBSCAN:

  • eps : la distance maximale entre deux observations leur permettant d'appartenir au mme voisinage,
  • min_samples : le nombre minimal de points appartennant au voisinage d'un point pour qu'il soit considr comme un "core point"

En effet, on a 3 types de points:

  • " core point " : tout point ayantau moins le nombre min_samples de points dans son voisinage,
  • " border point " : tout point qui n'est pas un "core point" mais qui a un "core point" dans son voisinage,
  • " noise point " : tout point qui n'est ni "core point" ni "border point" qui sera considr comme un bruit (outlier).

def delay_segment_dataframe(df, limit) :
    segnum = 0
    segments = []

    for i in range(df["time"].size) :
        if (df["delay"][i] > limit) :
            segments.append(segnum)
            segnum += 1;
        else :
            segments.append(segnum)

    df["segment"] = segments
    return df
